**Objects**
- An object represents a value and is automatically created by the interpreter when executing a line of code
- The garbage collection deletes unused objects

**Name Binding**
- Name binding is the process of associating names with interpreter objects

**Properties of Objects**
- Value - A value such as "20", "abcdef", or "55"
- Type - The type of the object, such as integer or string
- Identity - A unique identifier that describes the object
- *type()* returns the type of object
- Mutability indicates whether the object's value is allowed to be changed, integers and strings are immutable, which means they cannot be changed
- *id()* gives the value of an object's identity
